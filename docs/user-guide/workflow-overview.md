# 工作流程概述

本文档介绍 Panel Flow Refactor 的主要工作流程。

## 整体架构

Panel Flow Refactor 采用模块化设计，主要包含以下组件：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   输入层     │ -> │   处理层     │ -> │   输出层     │
└─────────────┘    └─────────────┘    └─────────────┘
```

## 工作流程阶段

### 1. 初始化阶段
- 加载配置文件
- 初始化面板数据
- 验证输入参数

### 2. 分析阶段
- 解析面板结构
- 识别依赖关系
- 评估优化空间

### 3. 重构阶段
- 应用重构策略
- 验证完整性
- 生成优化方案

### 4. 输出阶段
- 输出重构结果
- 生成报告
- 更新状态

## 模式选择

根据使用场景，可以选择不同的工作模式：

- **手动模式** - 精细控制每一步操作
- **自动模式** - 全自动执行重构流程

详情请参阅 [自动模式](./auto-mode.md)。
